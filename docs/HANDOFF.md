## Decisions

См. полный журнал в [DECISIONS.md](DECISIONS.md). Кратко:

- Python 3.11+.
- aiogram 3.x.
- Long polling, без webhook.
- База знаний — YAML/JSON в `data/`.
- Секреты — только через `.env`, не коммитятся.
- AI — отдельный слой, опционален, изолирован, не источник истины.
- Docker — позже, если останется время.
- Юридические тексты — справочные, с дисклеймером, без выдуманных норм.

## Next Step

**Stage 1 — базовый Telegram-бот с меню и статичной базой знаний.**

Конкретно для входящего агента (Codex):

1. Прочитать [AGENTS.md](../AGENTS.md), [TODO.md](TODO.md) (раздел Stage 1), [DECISIONS.md](DECISIONS.md), [ACCEPTANCE_CRITERIA.md](ACCEPTANCE_CRITERIA.md) разделы A, B, D, F.
2. Создать структуру:
   ```
   src/
     bot/__main__.py
     bot/handlers/
     bot/keyboards.py
     knowledge/
     config.py
   data/topics/*.yaml
   requirements.txt
   .gitignore
   ```
3. Добавить минимальные зависимости: `aiogram>=3.4`, `python-dotenv` (или `pydantic-settings`), `PyYAML`.
4. Реализовать `/start`, `/help`, главное меню (inline) и хендлеры всех 8 разделов с возвратом в меню.
5. Заполнить `data/topics/*.yaml` черновыми текстами (2–4 абзаца на раздел) — без выдуманных юридических норм, с дисклеймером в юридических разделах.
6. AI-режим **не делать** в Stage 1. Пункт «Спросить ИИ» либо отсутствует, либо отвечает «AI-режим выключен».
7. Добавить раздел «Как запустить локально» в README.
8. Smoke-тест вручную: `/start`, открыть каждый раздел, проверить кнопку «Назад».
9. Обновить этот HANDOFF.md и отметить Stage 1 в TODO.md.

Чего **не делать** на Stage 1: AI-интеграцию, Dockerfile, деплой, тесты сверх smoke-теста, расширение `SOURCES.md` (это Stage 2).
