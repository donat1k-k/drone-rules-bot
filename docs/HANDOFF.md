## Known Issues

- `docs/SOURCES.md` секции 1, 2, 3, 4 — все URL имеют статус «требует ручной проверки». Нужно открыть каждый и найти актуальный прямой URL перед защитой.
- Ограничение 4096 символов: все разделы в пределах нормы (проверить вручную при расширении контента).
- Текстовый поиск — keyword-матчинг без стемминга. Покрывает основные запросы; сложные перефразировки может не найти — это ожидаемо для простого поиска.

## Next Step

**Stage 3 — AI-режим.**

Переменные окружения для Stage 3:
- `OPENAI_API_KEY` — ключ API (OpenAI или совместимый провайдер)
- `OPENAI_BASE_URL` — base URL провайдера (для не-OpenAI провайдеров)
- `OPENAI_MODEL` — название модели
- `AI_ENABLED` — `true` / `false` (по умолчанию `false`)

Конкретно для входящего агента:

1. Прочитать [AGENTS.md](../AGENTS.md), [TODO.md](TODO.md) (раздел Stage 3), [ACCEPTANCE_CRITERIA.md](ACCEPTANCE_CRITERIA.md) раздел E.
2. Создать `src/ai/client.py` — обёртка над OpenAI-compatible API (библиотека `openai`).
3. Системный промпт: запрет выдумывать законы, требование опираться на базу знаний.
4. Загрузить содержимое YAML-топиков в системный промпт как контекст.
5. Добавить пункт «🤖 Спросить ИИ» в меню — только если `AI_ENABLED=true`.
6. Хендлер свободного ввода после нажатия кнопки AI.
7. Дисклеймер в каждом AI-ответе.
8. Graceful degradation: при ошибке провайдера — сообщение «попробуйте позже», бот не падает.
9. Добавить зависимость `openai` в `requirements.txt` (только если AI включён).

Чего **не делать** на Stage 3: Dockerfile, деплой, карту зон, погоду, NLP.
