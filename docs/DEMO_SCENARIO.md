# DEMO_SCENARIO — сценарий показа проекта

Короткий маршрут демонстрации работы бота преподавателю / комиссии / заказчику.
Длительность — около 3–5 минут. Лучше открыть бота заранее и держать диалог чистым.

---

## Перед показом

- [ ] Бот запущен (локально или на VPS).
- [ ] Telegram-аккаунт открыт, диалог с ботом найден.
- [ ] История диалога очищена (`/start` будет первым сообщением).
- [ ] Запасной канал: скриншоты в [docs/screenshots/](screenshots/) на случай проблем с сетью.

---

## Сценарий (7 шагов)

### 1. Запуск и главное меню

Отправь `/start`.
**Что показать:** приветствие, главное меню с 8 разделами + «🌦 Погода», «🧠 Мини-тест», (при `AI_ENABLED=true`) «🤖 Спросить ИИ».
**Подсказка для комиссии:** «Меню построено на inline-кнопках, навигация в одно касание».

### 2. Раздел базы знаний

Нажми «🛡 Правила безопасности».
**Что показать:** структурированный текст, кнопка «← Главное меню».
**Подсказка:** «Контент хранится в YAML-файлах в `data/topics/`. Обновлять можно без переписывания кода».

### 3. Юридический раздел с дисклеймером

Нажми «📋 Регистрация и закон».
**Что показать:** текст + блок дисклеймера в конце.
**Подсказка:** «Бот не даёт юридическую консультацию — только справочную информацию со ссылкой на Госуслуги / Росавиацию».

### 4. Погода для полёта

Из главного меню — «🌦 Погода для полёта» → ввести `Москва`.
**Что показать:** температура, ветер, осадки, светофор-оценка (✅ / ⚠️ / ⛔), дисклеймер.
**Подсказка:** «Open-Meteo API, бесплатный, без ключа. Логика оценки в `src/weather/client.py`».

Дополнительно (если есть время): ввести координаты `55.75, 37.61` — показать, что работают и они.

### 5. Текстовый поиск

Напиши в чат: `нужно ли регистрировать дрон`.
**Что показать:** бот находит раздел регистрации без меню.
**Подсказка:** «Keyword-поиск по полю `keywords` каждого YAML. Без NLP-библиотек».

### 6. Мини-тест

Из главного меню — «🧠 Мини-тест».
**Что показать:** 5 inline-вопросов с ✅/❌ после каждого, финальный счёт X/5.
**Подсказка:** «Тест из 5 вопросов по разным разделам — безопасность, регистрация, зоны, Return-to-Home».

### 7. Fallback + обратная связь

Напиши в чат что-то нерелевантное: `как купить пиццу`.
**Что показать:** сообщение «не нашёл раздел» + кнопки «✍️ Оставить вопрос» и «← Главное меню».
Нажми «✍️ Оставить вопрос», напиши тестовый текст → подтверждение «✅ Вопрос сохранён».
**Подсказка:** «Вопросы сохраняются в `logs/feedback.txt` — только текст и время, без персональных данных».

### Опционально — AI-режим

Если `AI_ENABLED=true` и ключ задан:
«🤖 Спросить ИИ» → `как откалибровать компас на дроне` → показать ответ с дисклеймером.
**Подсказка:** «AI работает через OpenAI-compatible API, системный промпт запрещает выдумывать законы».

## Что подчеркнуть в защите

- **8 разделов базы знаний** — все ключевые темы новичка.
- **Юридическая аккуратность** — везде дисклеймер, ссылки на официальные источники, не выдуманы статьи и штрафы.
- **Опциональный AI** — бот работает и без него, кнопка просто скрывается (graceful degradation).
- **Погода без платного API** — Open-Meteo бесплатный.
- **Деплой в одну команду** — `docker compose up -d --build`.
- **Минимум зависимостей** — `aiogram`, `aiohttp`, `pyyaml`, `python-dotenv`, `openai` (опц.).

## Возможные вопросы комиссии

| Вопрос | Короткий ответ |
|---|---|
| Где хранятся данные пользователей? | Нигде. Бот не ведёт БД пользователей. Только обратная связь — анонимный текст в `logs/feedback.txt`. |
| Что если выключить интернет AI-провайдеру? | Кнопка «🤖 Спросить ИИ» исчезает или ответ заменяется на «попробуй позже». Остальные функции работают. |
| Почему YAML, а не БД? | Контент стабильный, БД для учебного проекта избыточна. Решение зафиксировано в `docs/DECISIONS.md`. |
| Бот разрешает летать в Москве? | Нет. Бот **не выдаёт разрешений**. Только справка + ссылки на официальные сервисы. |
| Почему long polling, а не webhook? | Не нужен HTTPS-домен. Подходит для учебной демонстрации. |
