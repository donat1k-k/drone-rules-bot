# DECISIONS — технические решения

Журнал зафиксированных решений. Новые решения добавлять снизу с датой и обоснованием.
Формат записи: `## YYYY-MM-DD — Заголовок`, далее блоки **Context / Decision / Consequences**.

---

## 2026-05-23 — Язык: Python 3.11+

**Context.** Учебный проект, срок 1.5 дня. Нужен язык, на котором быстро поднимается Telegram-бот и есть зрелые библиотеки.
**Decision.** Python 3.11+.
**Consequences.** Большая экосистема, простые async-библиотеки. Минус — нужно следить за версией интерпретатора на деплое.

---

## 2026-05-23 — Фреймворк бота: aiogram 3.x

**Context.** Telegram Bot API, async, понятный роутинг, активно поддерживается.
**Decision.** aiogram 3.x.
**Consequences.** Современный API, удобные роутеры и фильтры. Не использовать примеры из aiogram 2 — они несовместимы.

---

## 2026-05-23 — Транспорт: Telegram Bot API (long polling)

**Context.** Демо для школьной защиты, временный деплой. Webhook требует HTTPS-домена и сложнее.
**Decision.** Long polling.
**Consequences.** Проще в запуске и отладке. На промышленный масштаб не годится, но для учебного — ок.

---

## 2026-05-23 — Хранение знаний: статичные файлы YAML/JSON

**Context.** Контент стабильный, обновлять его руками, БД пользователей не нужна.
**Decision.** Файлы в `data/` (предпочтительно YAML за читаемость; допустим JSON). Формат статьи: `id`, `title`, `body`, `disclaimer`, `sources`.
**Consequences.** Не нужна СУБД, миграции, ORM. Контент диффится в git.

---

## 2026-05-23 — Секреты: только в .env

**Context.** Токен бота и AI-ключи нельзя коммитить.
**Decision.** Все секреты — через переменные окружения, локально через `.env`. Шаблон — `.env.example`. `.env` — в `.gitignore`.
**Consequences.** На деплое переменные задаются через панель хостинга. Никаких хардкод-токенов.

---

## 2026-05-23 — AI-режим как отдельный изолированный слой

**Context.** AI должен помогать, но не быть источником истины и не ломать бот при отсутствии ключа.
**Decision.**
- AI-модуль — отдельный пакет `src/ai/`.
- Включается флагом `AI_ENABLED` + наличием API-ключа.
- Получает базу знаний как контекст.
- Системный промпт явно запрещает выдумывать юридические факты.
- Любая ошибка AI ловится и не валит остальной бот.

**Consequences.** Бот работает и без AI. На демо можно показать обе версии. Юридический риск ниже.

---

## 2026-05-23 — Docker: позже

**Context.** Срок 1.5 дня. Docker полезен, но не критичен для демо.
**Decision.** На Stage 1–3 не делаем Dockerfile. Если останется время — добавим на Stage 4.
**Consequences.** Деплой пойдёт через нативный Python runtime хостинга (Railway/Render/VPS с systemd).

---

## 2026-05-23 — Юридический контент: справочно + дисклеймер

**Context.** Школьники не должны брать на себя ответственность за юридическую консультацию.
**Decision.** Любой текст про законы/штрафы/зоны сопровождается дисклеймером и ссылкой на официальный источник. AI не имеет права давать конкретные правовые утверждения без опоры на базу знаний.
**Consequences.** Тексты получаются обтекаемее, но безопаснее. Часть точных норм заменяется на «уточняйте на …».
