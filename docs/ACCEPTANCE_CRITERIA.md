# ACCEPTANCE_CRITERIA — критерии готовности

Проект считается готовым к сдаче, если выполнены **все** пункты ниже.

## A. Базовая работоспособность

- [ ] Бот запускается одной командой из README на чистой машине с Python 3.11+.
- [ ] Токен бота читается из `.env`, в коде токена нет.
- [ ] `.env.example` присутствует и описывает все переменные.
- [ ] `.env` указан в `.gitignore`.
- [ ] При запуске бот отвечает на `/start` приветствием и показывает главное меню.
- [ ] Команды `/start`, `/help` работают.

## B. Контент / разделы

Все 8 базовых разделов доступны через меню и содержат осмысленный текст:

- [ ] Подготовка к полёту (чек-лист).
- [ ] Правила безопасности.
- [ ] Запрещённые зоны (с ссылкой на официальный сервис).
- [ ] Регистрация и закон (с дисклеймером).
- [ ] Потеря сигнала.
- [ ] Советы новичкам.
- [ ] FAQ (минимум 5 вопросов-ответов).
- [ ] Полезные ссылки (минимум 5, рабочие).

## C. Юридическая аккуратность

- [ ] В разделах «Запрещённые зоны», «Регистрация и закон», «FAQ» виден дисклеймер: информация справочная, требует проверки на официальных источниках.
- [ ] В тексте нет придуманных номеров статей, сумм штрафов, точных весов для регистрации, если они не подтверждены источником из [SOURCES.md](SOURCES.md).
- [ ] Все юридические утверждения сопровождаются ссылкой или формулировкой типа «по состоянию на …, уточняйте».

## D. UX

- [ ] В каждом подразделе есть кнопка возврата в меню.
- [ ] Тексты разделов не превышают разумный лимит Telegram-сообщения (4096 символов); длинные — разбиты на части.
- [ ] Стиль дружелюбный, без юридического канцелярита.
- [ ] Команда `/help` напоминает про дисклеймер.

## E. AI-режим (если включён)

- [ ] Включается только при наличии API-ключа в `.env` и `AI_ENABLED=true`.
- [ ] Если ключа нет — пункт меню «Спросить ИИ» отсутствует или возвращает сообщение «AI-режим выключен».
- [ ] Системный промпт явно запрещает выдумывать законы и требует ссылаться на базу знаний.
- [ ] AI получает базу знаний бота в качестве контекста.
- [ ] В каждом AI-ответе есть напоминание про справочный характер информации.
- [ ] Падение AI-провайдера не ломает остальной бот (try/except + сообщение «попробуйте позже»).

## F. Код и репозиторий

- [ ] Структура проекта понятна (`src/`, `data/`, `docs/`, `.env.example`, `README.md`).
- [ ] Нет закоммиченных секретов.
- [ ] `requirements.txt` (или `pyproject.toml`) фиксирует зависимости.
- [ ] Минимум 1 ручной smoke-тест описан в README (как проверить, что бот живой).

## G. Демонстрация

- [ ] Бот временно поднят на сервере и доступен по username в Telegram во время защиты.
- [ ] Можно показать комиссии: открыть, нажать каждый раздел, получить ответ.
- [ ] Есть короткая запись/скриншоты на случай, если интернет на защите подведёт.

## H. Документация

- [ ] README объясняет, что это и как запустить.
- [ ] `docs/HANDOFF.md` актуален и отражает реальный статус.
- [ ] `docs/DECISIONS.md` содержит все принятые технические решения.
- [ ] `docs/SOURCES.md` заполнен реальными ссылками.
