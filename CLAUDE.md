# CLAUDE.md — правила для Claude Code

Этот файл задаёт рамки работы Claude Code в проекте `drone-rules-bot`.
Читай его перед любой задачей. Если задача противоречит этим правилам — остановись и уточни у пользователя.

## 1. Что это за проект

Учебный школьный проект 11 класса: Telegram-бот «Правила управления дронами для начинающих».
Срок — около 1.5 дней. Цель — рабочий бот-инструктор для новичков, который можно показать комиссии и временно выложить на сервер.

Подробности — в [docs/PROJECT_BRIEF.md](docs/PROJECT_BRIEF.md).

## 2. Архитектура (целевая)

- **Язык:** Python 3.11+.
- **Фреймворк бота:** `aiogram` 3.x (Telegram Bot API).
- **База знаний:** статичные файлы (JSON или YAML) в каталоге `data/`. Никаких СУБД на Stage 1.
- **Слой AI:** опциональный модуль для свободных вопросов. Изолирован от ядра бота: ядро работает и без AI.
- **Конфигурация:** через переменные окружения (`.env`, библиотека `python-dotenv` или `pydantic-settings`).
- **Структура (целевая, на будущее):**
  ```
  src/
    bot/            # хендлеры, клавиатуры, роутеры
    knowledge/      # загрузка и выдача статей из data/
    ai/             # обёртка над AI-провайдером (опционально)
    config.py       # настройки из env
  data/
    topics/*.yaml   # статьи базы знаний
    faq.yaml
    links.yaml
  tests/            # минимально
  ```
- **Деплой:** локальный запуск + временный VPS / Railway / Render. Docker — позже, если останется время.

## 3. Стиль работы Claude в этом проекте

- Делай **минимум, который требует задача**. Не добавляй абстракций «на будущее».
- Не вводи новые зависимости без явной причины. Каждая новая зависимость — отдельное решение в [docs/DECISIONS.md](docs/DECISIONS.md).
- Не создавай админку, личный кабинет, БД пользователей, метрики, сложную NLP — это вне scope.
- Не делай «карту запретных зон» внутри бота. Только ссылки на официальные источники.
- Не пиши длинных docstring и комментариев «что делает код». Комментарий — только если поясняет *почему*.
- Не коммить токены и ключи. `.env` в `.gitignore`, шаблон — в [.env.example](.env.example).

## 4. Осторожность с юридической информацией

Это **критическое правило проекта**.

- Любой текст про законы, регистрацию, штрафы, запретные зоны — это **справочная информация**, не юридическая консультация.
- В каждом таком разделе бот должен напоминать пользователю: «информация справочная, проверяйте на официальных источниках».
- Не выдумывай номера статей, суммы штрафов, веса дронов для регистрации. Если точной нормы нет в [docs/SOURCES.md](docs/SOURCES.md) — пиши обобщённо и ссылайся на необходимость проверки.
- AI-режим **не должен выдумывать законы**. Системный промпт обязан запрещать конкретные юридические утверждения без ссылки на базу знаний.
- Если AI не уверен — он должен честно сказать «не знаю, проверьте на официальном сайте» и дать ссылку на раздел «Полезные ссылки».

## 5. Что Claude делает при ревью кода / диффов

- Проверяет, что токены/ключи не попали в код.
- Проверяет, что в обработчиках юридических тем есть дисклеймер.
- Проверяет, что AI-слой не вызывается там, где должен ответить статичный раздел базы знаний.
- Проверяет, что бот не падает при отсутствии AI-ключа (graceful degradation).
- Не одобряет PR с лишними фичами вне scope.

## 6. Что Claude НЕ делает в Stage 0

Сейчас (Stage 0) — только документация и рамки.
**Не писать**: код бота, `src/`, `requirements.txt`, Dockerfile, CI, тесты. Это придёт на Stage 1+.
