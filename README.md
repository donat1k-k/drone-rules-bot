# drone-rules-bot

Учебный проект 11 класса: Telegram-бот **«Правила управления дронами для начинающих»**.

## Что это

Telegram-бот-инструктор, который объясняет новичкам:

- подготовку к полёту;
- правила безопасности;
- запрещённые зоны (общие принципы + ссылки на официальные карты);
- регистрацию и закон (справочно);
- что делать при потере сигнала;
- советы новичкам;
- FAQ (11 вопросов и ответов);
- полезные ссылки.

Навигация по меню или через текстовый поиск — напиши вопрос текстом и бот найдёт подходящий раздел.

## Дисклеймер

Бот предоставляет **справочную** информацию. Это **не юридическая консультация**.
Все вопросы по регистрации, законам и запретным зонам нужно сверять с официальными источниками — см. [docs/SOURCES.md](docs/SOURCES.md).

## Технологии

- Python 3.11+
- aiogram 3.x (Telegram Bot API, long polling)
- База знаний в YAML (`data/topics/`)
- Простой текстовый поиск по ключевым словам (без NLP-библиотек)
- AI-режим через OpenAI-compatible API (библиотека `openai`)
- Погодная проверка через Open-Meteo API (бесплатно, без ключа)
- Конфигурация через `.env`
- Docker для деплоя на VPS

## Как запустить локально

**Требования:** Python 3.11+, токен бота от [@BotFather](https://t.me/BotFather).

```bash
# 1. Клонировать репозиторий
git clone <repo-url>
cd drone-rules-bot

# 2. Создать виртуальное окружение и установить зависимости
python -m venv .venv
# Windows:
.venv\Scripts\activate
# Linux/macOS:
source .venv/bin/activate

pip install -r requirements.txt

# 3. Настроить переменные окружения
cp .env.example .env
# Открой .env и вставь токен бота в TELEGRAM_BOT_TOKEN=

# 4. Запустить бота
python -m src.bot
```

Бот запущен — открой диалог с ним в Telegram и отправь `/start`.

## Погодная проверка условий полёта

Кнопка **«🌦 Погода для полёта»** доступна в главном меню без дополнительной настройки.

Введи **название города** или **координаты** в формате `55.75, 37.61`:

- Данные берутся из [Open-Meteo](https://open-meteo.com) — бесплатный сервис, ключ не нужен.
- Бот показывает температуру, скорость ветра и осадки, а также короткую оценку:
  - ✅ погодные условия выглядят спокойными для тренировочного полёта
  - ⚠️ условия требуют осторожности
  - ⛔ по погоде полёт лучше отложить
- Оценка — **справочная**. Она не является разрешением на полёт и не заменяет проверку запретных зон и правил.

## Запуск через Docker

**Требования:** Docker Desktop (Windows/Mac) или Docker Engine (Linux).

```bash
# 1. Убедись, что .env заполнен (скопируй .env.example и вставь токен)
cp .env.example .env

# 2. Собрать образ и запустить в фоне
docker compose up -d --build

# 3. Посмотреть логи (Ctrl+C — выйти из просмотра, бот продолжит работу)
docker compose logs -f

# 4. Остановить
docker compose down
```

Бот работает через long polling — webhook и домен не нужны. Переменные берутся из `.env` через `env_file`. При перезапуске контейнера (`restart: unless-stopped`) бот поднимается автоматически.

## Настройка AI-режима (опционально)

Бот поддерживает любой **OpenAI-compatible API** — OpenAI, [Polza AI](https://polza.ai) и другие.

В файле `.env` задай:

```env
AI_ENABLED=true
OPENAI_API_KEY=your_api_key_here
OPENAI_BASE_URL=https://api.polza.ai/v1   # для Polza AI; оставь пустым для OpenAI
OPENAI_MODEL=gpt-4o-mini                  # название модели провайдера
```

При `AI_ENABLED=false` или пустом ключе бот работает в обычном режиме — кнопка «🤖 Спросить ИИ» не показывается.

**Что умеет AI-режим:**
- отвечает на свободные вопросы о дронах;
- опирается на базу знаний бота как контекст;
- не выдумывает законы и штрафы — по юридическим вопросам отсылает на официальные источники;
- при ошибке провайдера возвращает сообщение «попробуй позже», бот не падает.

## Smoke-тест (ручная проверка)

1. `/start` → приветствие + главное меню с кнопками (включая «🌦 Погода для полёта»)
2. Нажать каждую кнопку → получить текст раздела
3. Нажать «← Главное меню» → вернуться в меню
4. Написать «нужно ли регистрировать дрон» → бот находит раздел регистрации
5. Написать «как купить пиццу» → fallback с предложением открыть меню
6. `/help` → текст содержит «написать вопрос текстом»
7. В разделах «Запрещённые зоны», «Регистрация», «FAQ» — виден дисклеймер
8. FAQ → 11 вопросов и ответов
9. «🌦 Погода для полёта» → ввести «Москва» → результат с оценкой и дисклеймером
10. «🌦 Погода для полёта» → ввести «55.75, 37.61» → результат по координатам
11. «🌦 Погода для полёта» → ввести «ываыва» → «Город не найден»
12. *(при AI_ENABLED=true)* Нажать «🤖 Спросить ИИ» → объяснение режима
13. *(при AI_ENABLED=true)* Написать «как правильно откалибровать компас» → ответ AI с дисклеймером
