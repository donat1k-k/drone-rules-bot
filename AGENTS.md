# AGENTS.md — правила для Codex и автономных агентов

Этот файл — инструкция для любого агента (Codex, фоновых задач, side-агентов), который реализует код в проекте `drone-rules-bot`.

## 1. Перед каждой задачей

Прочитай в указанном порядке:

1. [docs/HANDOFF.md](docs/HANDOFF.md) — текущее состояние проекта, где остановились.
2. [docs/TODO.md](docs/TODO.md) — какой Stage сейчас активен.
3. [docs/ACCEPTANCE_CRITERIA.md](docs/ACCEPTANCE_CRITERIA.md) — что считается готовым.
4. [docs/DECISIONS.md](docs/DECISIONS.md) — зафиксированные технические решения.
5. [CLAUDE.md](CLAUDE.md) — общие рамки и осторожность с законами.

Если что-то противоречит между файлами — приоритет: HANDOFF > DECISIONS > CLAUDE > остальное. Конфликт — остановись и спроси.

## 2. Как работать этапами

Работа разбита на Stages в [docs/TODO.md](docs/TODO.md). Базовая разбивка:

- **Stage 0** — документация и рамки. (✅ выполнено)
- **Stage 1** — базовый Telegram-бот: меню, статичная база знаний, основные разделы.
- **Stage 2** — FAQ, полезные ссылки, обработка потери сигнала и т.п. полные тексты статей.
- **Stage 3** — AI-режим для свободных вопросов (опционально).
- **Stage 4** — временный деплой, README с инструкциями, демо.

Правила:

- Делай **один Stage за раз**. Не прыгай вперёд.
- После Stage обнови [docs/HANDOFF.md](docs/HANDOFF.md): что сделано, что в полёте, следующий шаг.
- Если внутри Stage делаешь технический выбор не зафиксированный в DECISIONS — добавь запись в [docs/DECISIONS.md](docs/DECISIONS.md).

## 3. Что агент ДЕЛАЕТ

- Пишет минимально достаточный код под текущий Stage.
- Хранит секреты только в `.env` (локально) и читает через config-модуль.
- Добавляет дисклеймер к юридическим разделам бота.
- Логирует ошибки (минимально, `logging` стандартной библиотеки).
- Делает бот устойчивым к отсутствию AI-ключа: если `OPENAI_API_KEY`/`ANTHROPIC_API_KEY` пуст — AI-режим выключается, остальное работает.

## 4. Что агент НЕ ДЕЛАЕТ

- Не пишет код вне текущего Stage.
- Не вводит новые зависимости без записи в DECISIONS.
- Не коммитит `.env`, токены, ключи.
- Не выдумывает юридические нормы (статьи, штрафы, точные веса для регистрации). При сомнении — обобщай и ссылайся на проверку.
- Не строит карту запретных зон в боте. Только ссылки на официальные сервисы.
- Не делает админку, БД пользователей, личный кабинет, метрики, сложную NLP.
- Не использует AI там, где есть статичный раздел базы знаний.
- Не пушит и не деплоит без явной просьбы пользователя.
- Не правит этот файл и `CLAUDE.md` без согласования.

## 5. Контракт с базой знаний

База знаний — единственный источник правды для бота на Stage 1–2.

- Файлы в `data/` (YAML или JSON).
- Каждая статья: `id`, `title`, `body`, `disclaimer` (для юридических — обязателен), `sources` (список URL из [docs/SOURCES.md](docs/SOURCES.md)).
- Если AI-режим включён — он получает базу знаний как контекст и **обязан опираться на неё**. Системный промпт явно запрещает придумывать факты.

## 6. Что сдавать в конце Stage

- Рабочий код, который запускается одной командой (`python -m bot` или аналог).
- Обновлённый `HANDOFF.md`.
- Обновлённый `TODO.md` (отметить выполненный Stage).
- Если появились новые решения — запись в `DECISIONS.md`.
- Краткий итог в чате: что сделано, как запустить, что не успели.

## 7. Если что-то непонятно

Останавливайся и спрашивай. Учебный срок 1.5 дня — лучше уточнить, чем переделывать.
